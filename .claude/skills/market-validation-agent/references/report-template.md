# Report Template

The exact markdown structure for the final validation report. Follow this template — consistency across validations is part of the value, and downstream readers (investors, advisors, the user themselves rereading later) rely on knowing where to find things.

Save the file as `/mnt/user-data/outputs/<short-idea-slug>-validation.md` and present it with `present_files`.

## Notes on filling the template

- **Idea slug**: convert the idea name to lowercase-kebab-case for the filename (e.g. "ai pet trainer" → `ai-pet-trainer-validation.md`).
- **No empty sections**: if a section genuinely has no content (e.g. regulatory is fully background), write one sentence stating so rather than leaving a placeholder.
- **Numbers in headlines**: bold the headline number in each section so a skim-reader can find the takeaway in 30 seconds.
- **Sources discipline**: every claim that depends on web-sourced data must point to a source — either inline as `[source]` linked text or referenced by number from the sources list. Both styles are acceptable; do not mix within one report.
- **Length target**: 2,500–5,000 words for the full deep-dive. Shorter when evidence is thin; longer for complex multi-segment markets.
- **Tone**: professional, evidence-led, honest. Not promotional. Not artificially hedged. If a finding is negative, state it clearly.
